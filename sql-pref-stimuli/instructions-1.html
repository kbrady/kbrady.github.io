Basic Select Statement

For this section we will use the participant table in the SQL interpreter as an example. The data in this table is based on the Milgram Obedience Experiment. In this experiment, each participant was told to administer a shock to motivate another participant to answer questions correctly. The other participant was actually an actor pretending to be shocked, and the goal of the experiment was to test whether participants would obey the experimenter and administer the maximum shock of 450 volts. In one condition, the experimenter wore a lab coat, so participants perceived them as an authority figure. In another condition, the lab coat wearing "experimenter" was called away and replaced by someone wearing ordinary clothes, whom the participants would perceive as having less authority.

Each participant corresponds to a row in the participant table and has attributes id, maximum_shock, age and experimenter. id is the number used to identify each participant in the study. maximum_shock is the maximum number of volts the participant thought they were administering to the actor during the study. age is the age of the participant when the study was conducted. experimenter tells us whether the participant was in the condition with a lab coat wearing experimenter or the ordinary clothes wearing experimenter.

Here are the first few rows of the participants table

id maximum_shock age experimenter
1 450 29 Lab Coat
2 420 21 Ordinary Clothes
3 360 26 Lab Coat

The first thing we will do in SQL is look at all the participant ids to see how many data points we have. In the SQL interpreter type: SELECT id FROM participants;

Hit enter when you are done typing. You should see a list of 40 ids ranging from 1 to 40. If you got an error message which says "no table named X" it is either because you miss typed participants or you incorrectly capitalized the table name.

Now let us break down what is going on in the code you just typed. The first word SELECT tells SQL that you want it to give you some values. In addition to selecting data, SQL allows users to insert information into the table, delete information from the table and create and delete entire tables. This tutorial is focused on viewing data which is in the database, so we wonâ€™t worry about those other commands.

The second part of the phrase, id, tells SQL what column in the table you want to select. In this case we wanted to view only the participant ID column.

The third word, FROM, tells SQL that we are done listing columns to select and now want to tell it which table we are selecting these columns from.

The final word in our statement, participant, is the name of the table we would like to select from.

The semicolon at the end of the statement is very important. If you run the statement without it, you will get a prompt to enter more information.

Now that we know how to select one column, let us write a statement to select more than one. We can do this by listing the columns we want with commas between each column name. For instance if we would like to get the maximum shock that each participant gave in addition to their id, we can do this by running: SELECT id, maximum_shock FROM participants;

The order of the list determines the order of the columns in the output. Thus if we want the maximum shock to be the first column in the table and id to be the second column, we use the command: SELECT maximum_shock, id FROM participants;

There are a few aggregation functions built into the SQL framework for calculating statistics on fields in the database. For instance, if we want to know the highest maximum shock that any participant administered, we can run the following command: SELECT MAX(maximum_shock) FROM participants;

If we would like to know the minimum maximum shock, we can run the command: SELECT MIN(maximum_shock) FROM participants; When using aggregation functions, we must give instructions on how to aggregate every column which is selected. Otherwise, SQL will randomly choose a value from the column. As an example, compare the outputs of the following commands: SELECT MIN(maximum_shock), id FROM participants; SELECT MIN(maximum_shock), MIN(id) FROM participants; SELECT MIN(maximum_shock), MAX(id) FROM participants;

Other functions you might find useful are COUNT, which counts the number of rows in the results and AVG which gives the average value of a column. You can try these out by running: SELECT COUNT(maximum_shock) FROM participants; SELECT AVG(maximum_shock) FROM participants;
